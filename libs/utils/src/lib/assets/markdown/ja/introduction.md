# 紹介

ここでは、TagCheck の概要と、このアプリケーションの使い方を確認できます。

## はじめに

このセクションでは、利用開始に必要な基本手順を案内し、TagCheck が Google Tag Manager（GTM）のタグ監査プロジェクト管理をどのように支援するかを説明します。

## プロジェクト

このセクションでは、プロジェクトの作成と管理の基本を説明します。新しいプロジェクトの作成、タグの追加、監査の実行に加えて、編集や削除などの管理機能も扱います。

## 設定

このセクションでは、必要に応じてアプリケーション設定を調整する方法を説明します。内容は次のとおりです。

- プロジェクト情報
- 認証
- プリロードデータ
- Browser Arguments
- Google Tag Manager とリクエスト傍受の設定
- プロジェクトの入出力

## TagCheck の仕組み

以下では、TagCheck の基本的な動作を簡潔に説明します。

## Puppeteer

Puppeteer は Node.js のライブラリで、DevTools Protocol を通じてヘッドレス Chrome や Chromium を操作する高水準 API を提供します。TagCheck はこれを使ってブラウザ自動化による監査を行い、品質保証の要件に応じて複数のブラウザインスタンスを扱うこともできます。

## Chrome Recorder

Chrome Recorder は、ブラウザ上のユーザー操作を記録し、Puppeteer スクリプトを生成するツールです。TagCheck は Recorder の JSON 出力を利用して操作を再生し、Puppeteer で監査を実行します。

## 監査パイプライン

ユーザーが監査を開始すると、アプリケーションは新しいブラウザインスタンスを起動して記録済みの操作を再生します。その後、タグが正しく発火しているかを確認し、結果レポートを生成します。レポートには、正しく発火しなかったタグとその理由が含まれます。
