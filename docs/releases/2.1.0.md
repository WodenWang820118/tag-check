# Tag Check 2.1.0

Tag Check 2.1.0 prepares the desktop release flow for repeatable publishing by
making `VERSION` the canonical source for release metadata and keeping the
desktop manifests aligned with it.

## Highlights

- Desktop release metadata now resolves from a single `VERSION` file and checks
  that `package.json`, `tauri.conf.json`, and `Cargo.toml` stay synchronized.
- The release tooling exposes explicit sync and check commands so version drift
  can be corrected locally before a publish workflow reaches `main`.
- Recent backend coverage stabilization and product-doc translation fixes remain
  included in this release line.

## Desktop Assets

- Windows: `tag-check-desktop-v2.1.0-windows-x64-nsis.exe`
- macOS: `tag-check-desktop-v2.1.0-macos-arm64-unsigned.tar.gz`
- Linux: `tag-check-desktop-v2.1.0-linux-x64-appimage.AppImage`

## Operator Notes

- Update the root `VERSION` file first when preparing the next stable release.
- Run `pnpm run version:sync` before committing a version bump.
- Run `pnpm run version:check` or `pnpm run tauri:release -- describe` to
  verify release metadata before pushing to `main`.
