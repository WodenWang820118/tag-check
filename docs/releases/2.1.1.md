# Tag Check 2.1.1

Tag Check 2.1.1 tightens the desktop release preparation path so version drift
can be corrected before the publish workflow reaches the release gate.

## Highlights

- The release version sync command now resolves the latest stable semver across
  `VERSION`, `package.json`, `tauri.conf.json`, and `Cargo.toml`, then writes the
  resolved version back to all four release authorities.
- Version comparison now uses numeric semver ordering, so versions such as
  `2.10.0` correctly sort after `2.9.9`.
- Local release-tooling coverage now verifies malformed authority recovery,
  explicit sync targets, and the required release notes file for the checked-in
  `VERSION`.

## Desktop Assets

- Windows: `tag-check-desktop-v2.1.1-windows-x64-nsis.exe`
- macOS: `tag-check-desktop-v2.1.1-macos-arm64-unsigned.tar.gz`
- Linux: `tag-check-desktop-v2.1.1-linux-x64-appimage.AppImage`

## Operator Notes

- Run `pnpm run version:sync` after changing any release version authority.
- Run `pnpm run version:check` or `pnpm run tauri:release -- describe` before
  pushing a release PR.
- Add `docs/releases/<version>.md` whenever the checked-in `VERSION` advances.
