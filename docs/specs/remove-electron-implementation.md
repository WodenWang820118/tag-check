# Spec: Remove Legacy Electron Implementation

- **Status:** Draft
- **Author:** Codex
- **Last Updated:** 2026-04-22
- **Related Issues:** User request to fully remove Electron implementation now that Tauri is the desktop runtime

---

## 1. Objective

Remove the remaining Electron-specific implementation, tooling, and documentation from the workspace so the desktop app path is unambiguously Tauri-only. The goal is to eliminate stale desktop entrypoints, packaging scripts, Electron-only tests, and misleading docs/config while preserving the current Tauri-based desktop workflow.

## 2. Technical Design

### Assumptions

- `apps/desktop-tauri` is the only supported desktop application project.
- Electron code under the repo root `src/` is legacy and no longer used by production or development flows.
- Desktop packaging and release should use the existing Tauri targets and helper script `tools/scripts/tauri/prepare-runtime.ts`.
- Web and backend Nx projects must continue to build and test without Electron installed as a runtime dependency.
- Electron-related historical references will only be cleaned in actively maintained files touched by this task; broader prose archaeology is out of scope.

### Tech Stack

- **Language/Framework:** TypeScript, Angular, NestJS, Nx, Tauri
- **Desktop Packaging:** `@tauri-apps/cli` through `desktop-tauri` Nx targets
- **Key Libraries/Dependencies:** Tauri CLI, Playwright, Vitest

### Commands

- **Workspace status:** `pnpm review:status`
- **Desktop dev:** `pnpm nx run desktop-tauri:dev`
- **Desktop build:** `pnpm nx run desktop-tauri:build`
- **Desktop bundle:** `pnpm nx run desktop-tauri:bundle`
- **Unit tests:** `pnpm test -- --runInBand` or targeted Vitest commands as appropriate

### Project Structure

```text
apps/
  desktop-tauri/                  <-- Keep as the only desktop app
    src-tauri/
docs/
  specs/
    remove-electron-implementation.md
tools/scripts/
  tauri/
    prepare-runtime.ts            <-- Keep and preserve behavior
src/
  main.ts                         <-- Remove legacy Electron main process
  preload.ts                      <-- Remove legacy Electron preload
  renderer.ts                     <-- Remove legacy Electron renderer notes
  frontend.ts                     <-- Remove legacy Electron window bootstrap
  backend.ts                      <-- Remove legacy Electron backend bootstrap
  logger.ts                       <-- Remove Electron logging wrapper if unused
src/e2e/
  app.e2e-spec.ts                 <-- Remove Electron-only Playwright flow
```

### Code Style & Patterns

Changes should be surgical:

- delete legacy Electron-only source files instead of leaving dead code
- replace Electron-specific script names, release steps, and comments with Tauri-aligned wording
- keep Nx/Tauri entrypoints explicit rather than adding compatibility shims

Representative pattern:

```json
{
  "scripts": {
    "dev-tauri": "nx run desktop-tauri:dev",
    "build-tauri": "nx run desktop-tauri:build",
    "bundle-tauri": "nx run desktop-tauri:bundle"
  }
}
```

### Cleanup Scope

The implementation should explicitly audit and update or remove these Electron-era surfaces:

- root `package.json` scripts and Electron dependencies
- `pnpm-lock.yaml` entries regenerated by dependency removal
- `forge.config.mjs`
- `forge.env.d.ts`
- `vite.main.config.ts`
- `vite.preload.config.ts`
- `vite.renderer.config.ts` audit for Electron-era plugins or assumptions
- `playwright.config.mjs` Electron project wiring
- `src/e2e/app.e2e-spec.ts`
- root `src/` Electron process/bootstrap files
- `tsconfig.base.json` and related `tsconfig.*.json` includes/excludes referencing removed Electron files
- `eslint.config.mjs` overrides or ignores tied to Electron files
- `index.html` Electron-era renderer assumptions, if any
- `electron-window.png` stale Electron artifact
- `.github/workflows/release.yaml` and any other actively maintained CI/release workflow that still invokes Electron Forge
- actively maintained docs such as `README.md` and `CONTRIBUTING.md`

## 3. Testing Strategy

- **Unit Tests:** Keep existing `tools/scripts/tauri/prepare-runtime.test.ts` passing because it protects the current Tauri runtime preparation flow.
- **Integration Tests:** Validate changed build or packaging scripts by running the relevant Tauri/Nx targets or targeted dry-run-safe commands where possible.
- **End-to-End (E2E) Tests:** Remove Electron-only Playwright coverage and ensure the shared Playwright config no longer advertises an Electron project.
- **Dependency Verification:** Reinstall workspace dependencies after removing Electron packages and confirm the lockfile resolves cleanly.
- **CI/Release Verification:** Inspect release workflow changes to confirm no maintained job still invokes Electron Forge or Electron-only publish/package commands.
- **Configuration Verification:** Audit Vite, TypeScript, ESLint, and root HTML/static assets for references to deleted Electron files or Electron-only behavior.

## 4. Boundaries & Non-Goals

- Do not redesign the Tauri desktop architecture.
- Do not change frontend or backend product behavior unrelated to Electron removal.
- Do not add a new Tauri E2E suite in this task unless required to keep existing checks green.
- Do not refactor unrelated build scripts or documentation beyond Electron-to-Tauri cleanup.
- Do not perform a repo-wide sweep of historical comments beyond files already touched for functional cleanup.

## 5. Success Criteria

- [ ] No maintained npm script still starts, packages, or publishes Electron.
- [ ] Electron-related dependencies are removed from `package.json`, and `pnpm-lock.yaml` is updated accordingly.
- [ ] Legacy Electron root-process source files are removed from the repo if unused.
- [ ] Playwright configuration and desktop-related tests no longer reference Electron.
- [ ] Release/build configuration no longer points to Electron Forge.
- [ ] Maintained CI/release workflows no longer invoke Electron-specific commands.
- [ ] User-facing documentation no longer instructs contributors to use Electron flows where Tauri is now canonical.
- [ ] Existing Tauri runtime preparation tests continue to pass.

## 6. Open Questions

- None blocking. Historical Electron prose cleanup is limited to actively maintained files touched by this change.
